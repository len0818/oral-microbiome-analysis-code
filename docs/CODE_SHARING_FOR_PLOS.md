# PLOS ONE code sharing checklist

Use this repository as the **public code release** for the manuscript.

## Confirm before submission

- [ ] All author-generated code underlying the manuscript findings is included.
- [ ] The code is publicly available without restriction.
- [ ] A repository URL is available.
- [ ] A persistent archive DOI has been created, for example with Zenodo.
- [ ] An open-source license is included.
- [ ] The README explains dependencies and command-line usage.
- [ ] The manuscript Data Availability Statement includes the code repository link and DOI.
- [ ] If the underlying data are not in this repository, the manuscript separately explains the data-access conditions.

## Notes for this package

- This package intentionally excludes raw data.
- This package intentionally excludes exploratory notebooks that contained local working paths.
- The public release keeps only cleaned executable scripts and documentation.
